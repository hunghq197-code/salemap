### 2. Audit Findings

Recommended follow-ups:

- Production Google Maps fallback:
  - `lib/providers/maps/google-maps.ts` returns demo places/routes when `GOOGLE_MAPS_API_KEY` is missing.
  - This is useful for staging/dev but risky in production because users may see demo leads as if they were real.
  - Recommended change later: only allow demo maps when `NODE_ENV !== "production"` or a dedicated flag such as `ENABLE_DEMO_MAPS=true` is set.
- Same-origin/rate-limit consistency:
  - Some mutating API routes have auth/ownership checks but do not consistently call `enforceSameOrigin` or `rateLimit`.
  - Examples noted during audit included payment request PATCH, export leads, import execute/mapping/validate, bulk actions, and AI save routes.
  - Recommended later: add a small shared guard for authenticated mutate routes.
- Repo hygiene:
  - `.npm-cache/` was generated by local npm usage and initially appeared as untracked files.
  - `.npm-cache/` was later added to `.gitignore`.
