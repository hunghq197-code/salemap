# SaleMap Staging Beta QA

## Mục Tiêu Test Beta

- Kiểm tra user có hiểu SaleMap dùng để làm gì không.
- Kiểm tra flow tạo lead/note/reminder.
- Kiểm tra import có dễ dùng không.
- Kiểm tra pipeline có hữu ích không.
- Kiểm tra mobile có dùng được ngoài thị trường không.

## Checklist Test 30 Phút

1. Register.
2. Onboarding.
3. Tạo lead thủ công.
4. Thêm note.
5. Tạo follow-up/reminder.
6. Import CSV mẫu.
7. Mở pipeline.
8. Đổi status lead.
9. Xem analytics.
10. Gửi feedback.

## Feedback Questions

1. Bạn có hiểu SaleMap dùng để làm gì trong 1 phút đầu không?
2. Bạn có tạo lead đầu tiên dễ không?
3. Bạn có thấy reminder/follow-up hữu ích không?
4. Bạn có muốn import dữ liệu cũ không?
5. Pipeline có giống cách bạn chăm sóc khách không?
6. Tính năng nào bạn thấy đáng trả tiền nhất?
7. Điểm khó chịu nhất là gì?
8. Bạn có tiếp tục dùng app trong 7 ngày tới không?

## Cách Ghi Nhận Kết Quả

- Mỗi beta user tạo một dòng tracking riêng: email test, ngày mời, trạng thái test, lỗi gặp, feedback chính.
- Không yêu cầu user nhập dữ liệu khách hàng thật/nhạy cảm trong 3-7 ngày đầu.
- Lỗi crash, login, mất data, lộ data, payment nhầm trạng thái là P0/P1.
- Feedback UX/copy/flow là P2/P3 nếu không chặn test.

## Exit Criteria Sau 5-10 Users

- Ít nhất 5 user hoàn tất register/onboarding.
- Ít nhất 5 user tạo lead đầu tiên.
- Ít nhất 3 user tạo note/reminder.
- Ít nhất 2 user thử import.
- Không có lỗi P0/P1 còn mở.
- Có đủ feedback để quyết định sprint tiếp theo.
